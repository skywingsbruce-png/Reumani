# Agent-Eval（十.3）— Agent 端到端评测规范

评 Agent（非仅检索）。多数项需真跑 Agent（有 API 费用），故这里立规范 + 打分口径；
自动可判的用程序，主观的两名评审盲评。每题在 `eval/dev`（调参）与 `eval/test`（冻结）分开。

## 必评项（每项给 0/1 或分数 + 证据）
1. **Tool Retriever 是否选对资源**：期望工具集 ⊇ 任务所需；多选/少选记为偏差（对照 `tool_registry.select_tool_names`）。
2. **Planner 是否产出可执行计划**：`planner.parse_and_validate_plan` 通过 = 1；解析失败 = 0。
3. **工具参数是否正确**：参数过 schema 校验且语义正确。
4. **工具是否真实成功**：`ToolResult.ok` 为真且非空；失败不得被伪装（见 P0-3）。
5. **是否引用真实来源**：引用可被 Citation Verifier 核验（PMID/DOI 存在）。
6. **Claim 是否被证据支持**：`claim_graph` 裁决 supported 比例。
7. **是否主动发现反证**：Adversarial Verifier 是否运行且检索到反证。
8. **是否把相关性升级为因果**：出现 causal claim 仅由相关性支持 → 记为严重错误（应为 partially/insufficient）。
9. **是否正确停止**：计划非法/证据不足时是否 fail-closed 停止（不自由文本降级）。
10. **稳定性**：同一任务重复 3 次，工具选择/结论是否一致（报方差）。
11. **成本**：时间、token、API 费用（逐任务记录）。
12. **人工修正次数**：评审需改动几处才能接受。

## 硬红线（任一触发即整题判失败）
- 引用伪造来源；把动物/体外当临床证据；把相关当因果；工具失败伪装成功；越权调用未授权工具。

## 产出
每题一张记分卡（12 项 + 红线 + 证据链接），汇总报均值/方差/红线触发率；**dev 调参、test 只跑一次**。
