# 证据卡片 + 验证（循证接地）

这是修补"只看标题猜证据"这个盲区的技能。原来查文献只有标题，现在能拿到摘要并结构化。

## 什么时候用
- 要**判断证据强度**（RCT vs 横断面 vs 动物实验）、**样本量**、研究设计 → 用证据卡片，别只看标题。
- 给一个结论/研究方向**把关**，看它是不是被真实文献支持、有没有过度解读 → 用验证。

## 工具（agent 可直接调用）
- `search_evidence(query, n=8)`：检索 → 抓摘要 → 生成结构化证据卡片。
- `verify_claim(claim, query)`：针对某论断检索证据并核对（支持/部分支持/不支持 + 是否过度解读）。

## 典型用法
1. 用户问"某方向证据够不够强" → `search_evidence` 出卡片 → 按证据强度排序解读。
2. 辩论/裁决给出结论后 → `verify_claim("CD19-CAR T 能逆转 SSc 纤维化", "CAR T systemic sclerosis fibrosis")`
   → Verifier 核对：支持它的是几篇？是不是小样本单臂被当成强因果？

## 原则
- 只依据真实摘要，卡片里"未说明"就是摘要没写，不许编。
- 证据强度是辅助判断，最终学术结论仍由研究者负责。
- 全文级细节（完整 Methods、样本量细分）摘要可能不全，必要时读原文 PDF。
